# Environment Contract

Carbide separates local development placeholders from real deploy secrets.

Local development uses Docker Compose defaults so `carbide run dev` works from
a clean checkout. Real deployment secrets belong to the deployment or IaC layer,
not to a separate secrets container or secrets sidecar container.

## Files

- `config/env.schema.json` is the source of truth for required, optional,
  secret, browser-exposed, and framework-owned variables.
- `.env.example` shows local overrides that may be copied into `.env`.
- `.env` is ignored and must not be committed.
- `docker-compose.yml` owns local default values.

## Rules

- Do not print secret values in logs, CLI output, docs, or errors.
- Browser-exposed variables must not be secret.
- Framework-owned variables are generated by Carbide or Compose and should not
  be casually overridden by app code.
- Deploy targets must provide real secret values through their own secret
  manager or IaC provider.

## Checks

Run:

```sh
carbide doctor env
```

The command validates the schema, reports missing required variables, and
summarizes secret/browser/framework-owned counts without printing values.
