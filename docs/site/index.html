Docs Prologue

Installation

Install the Carbide CLI, create an app, and start the local Docker stack. The generated app begins with a Bun frontend, Go API, and Postgres database.

Start

Paste this into your AI agent:

I'm building a Carbide app.
Fetch and follow the instructions at https://carbide.ryangerardwilson.com/for/agents.
Treat that Markdown as the source of truth for Carbide setup and project work in this session.

The full local walkthrough lives in Create Your First App.

Runtime Overview

The browser talks to the web service. The web service proxies API traffic to the Go API service. The API service owns database access through the Postgres service.

Browser Uses the host URL printed by the CLI.
Web Bun, React, Tailwind, static assets, and API proxy.
API Go HTTP routes, auth, sessions, and JSON responses.
DB Postgres schema, users, sessions, and migrations.

Product Contract

  • Apps run through a Bun/React/Tailwind web container, Go API container, and Postgres db container.
  • Postgres is mandatory, not one adapter among many.
  • The Bun web service is the public entrypoint and proxies /api to the API service.
  • Infrastructure belongs in version-controlled code.
  • Environment and secret contracts are checked in and validated without printing values.
  • Runtime versions are explicit baselines with digest-pinned images and exact framework-owned package versions.
  • carbide health validates project shape, required config, Compose, env/secrets, and legacy-regression markers.
  • Deploys follow preview before apply; targets are environments that can start as one host and grow into web/API/db roles across servers.
  • Deployment covers single-VM ssh-compose targets and multi-VM role topology in carbide.toml.
  • The default starter opens account creation first, then supports login, logout, and dashboard.
  • The first IaC target is generated Docker Compose for local development.
  • The CLI is implemented in Go and renders colored, aligned, script-safe output with full-width per-container startup and shutdown animation, status tables, timestamped logs, live log follow, and queryable structured dev logs.

First Milestone

  1. Boot with one command.
  2. Serve the React app through Bun.
  3. Proxy /api to the Go API service.
  4. Create the first user with a Postgres-backed session.
  5. Open the dashboard.
  6. Validate the app laws with carbide health.
  7. Run Docker-backed health/auth checks with carbide health runtime.

Current Automation

The repo has a CI gate for contract checks. The documentation app serves the static site from docs/site through the checked-in Carbide docs app.

The published docs domain is carbide.ryangerardwilson.com.
