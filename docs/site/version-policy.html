Docs Prologue

Version Policy

Carbide treats versions as explicit baselines. The durable product is the generated app contract, not a moving dependency list.

Current Baseline

  • Go modules: go 1.25.0
  • Go builder: golang:1.26-bookworm@sha256:b305420a68d0f229d91eb3b3ed9e519fcf2cf5461da4bef997bf927e8c0bfd2b
  • API runtime: debian:trixie-slim@sha256:28de0877c2189802884ccd20f15ee41c203573bd87bb6b883f5f46362d24c5c2
  • Bun: oven/bun:1.3.14-debian@sha256:9dba1a1b43ce28c9d7931bfc4eb00feb63b0114720a0277a8f939ae4dfc9db6f
  • React: 19.2.7
  • Tailwind CSS: 4.3.2
  • Postgres: postgres:17-alpine@sha256:dc17045ccfd343b49600570ea734b9c4991cf1c3f3302e67df51e3b402dd55c4

Rules

  • No latest, semver ranges, or floating Docker image tags for framework-owned runtimes.
  • Docker references include @sha256: digests.
  • carbide doctor checks the local project baseline.
  • The scheduled dependency audit reports drift but never edits files.
  • Postgres major-version changes require migration notes and explicit operator action.

Upgrade Stance

Framework-owned upgrades should be explicit product decisions. A dependency bump must preserve the generated app contract, update the baseline, and pass doctor, scaffold, smoke, and docs regression checks.
