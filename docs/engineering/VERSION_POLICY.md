# Carbide Version Policy

Carbide is meant to make generated apps durable, not frozen. The eternal part
of Carbide is the app contract: one `web` service, one `api` service, one
`db` service, checked-in infrastructure, checked-in environment rules, explicit
runtime boundaries, and `carbide doctor` as the local contract check.

Runtime versions are implementation baselines. They are pinned, audited, and
upgraded deliberately.

## Current Baseline

New projects generated by this repository use:

| Runtime | Baseline |
| --- | --- |
| Go module directive | `go 1.25.0` |
| Go builder image | `golang:1.26-bookworm@sha256:b305420a68d0f229d91eb3b3ed9e519fcf2cf5461da4bef997bf927e8c0bfd2b` |
| API runtime image | `debian:trixie-slim@sha256:28de0877c2189802884ccd20f15ee41c203573bd87bb6b883f5f46362d24c5c2` |
| Bun image | `oven/bun:1.3.14-debian@sha256:9dba1a1b43ce28c9d7931bfc4eb00feb63b0114720a0277a8f939ae4dfc9db6f` |
| React | `19.2.7` |
| Tailwind CSS | `4.3.2` |
| Postgres | `postgres:17-alpine@sha256:dc17045ccfd343b49600570ea734b9c4991cf1c3f3302e67df51e3b402dd55c4` |

The generated `carbide.toml` records this same baseline under `[runtime]`.

## Rules

- Generated projects must not use `latest`, semver ranges, or floating Docker
  image tags for framework-owned runtimes.
- Docker references must include an `@sha256:` digest.
- Package versions for React, React DOM, Tailwind CSS, and the Tailwind CLI
  must be exact versions.
- Generated Go modules use a supported Go directive, currently `go 1.25.0`,
  while the containerized builder uses the current Go 1.26 toolchain.
- Postgres major-version changes are framework baseline changes, not casual
  dependency bumps. Existing volumes need migration or recreation.
- Bun remains behind the `web` container boundary. Carbide should avoid
  depending on Bun-only APIs outside that service contract.

## Upgrade Process

1. Run the scheduled dependency audit or `bash tests/contract/audit_versions.sh`
   to see newer stable versions and changed image digests.
2. Update this policy, `scaffold/carbide.toml`, Dockerfiles, Compose, lockfiles,
   docs, and contract tests in the same change.
3. Run `carbide doctor framework` from the framework checkout.
4. Publish migration notes when a baseline changes database major versions,
   Go directives, or container operating systems.

The audit job reports drift. It does not edit files, run package upgrades, or
mutate generated projects.
