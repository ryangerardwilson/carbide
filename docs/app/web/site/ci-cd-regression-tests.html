CI/CD Regression Plan

The test system starts with repository contracts and grows into Go API, Postgres, container, IaC, security, and documentation regression suites.

The regression plan is runner-agnostic: run it locally, wire it into any CI system, or both.

Required Gates

Change Gate

  • Repository contract checks.
  • Non-mutating dependency and image drift audit.
  • Go CLI unit tests.
  • Shell syntax checks for repo-owned test and launcher scripts.
  • Documentation site contract checks.
  • Generated Docker stack smoke test with registration-first, Postgres-backed auth.
  • carbide health fast app-law checks.
  • Future API unit, integration, and compatibility checks.

Publish Gate

  • The change gate.
  • Documentation deployment through the Carbide docs app.
  • Extensionless documentation routes with redirects from legacy .html URLs.
  • Future release candidate smoke checks.

Release Gate

  • Supported Go version matrix.
  • Race and compatibility matrix.
  • Postgres integration matrix.
  • Generated project smoke tests.
  • Migration up/down tests.
  • Container build and boot tests.
  • Published docs verification after the app-owned deploy script runs.

Regression Suites

  • Repository contract: protects required files, docs, runner-agnostic regression hooks, product constraints, and the 1000-line limit for checked source and test files.
  • CLI scaffold: tests the Go CLI, shared output renderer, carbide new, carbide init, the staged carbide audit / carbide resolve / carbide fix flow, compact run dev output, status table rendering, detach-without-teardown on Ctrl+C, idempotent clean dev normalization, explicit stop dev teardown, per-container progress behavior, timestamped service-log streaming, live log follow, structured log querying, placeholder replacement, and empty-directory enforcement.
  • API build and behavior: protects generated API builds, public API behavior, and explicit release compatibility work.
  • Unit tests: protects router, request, response, middleware, config, and logging behavior.
  • Compatibility and race tests: exposes concurrency and generated app compatibility failures.
  • Postgres integration: tests readiness, pooling, migrations, transactions, and parameterized queries.
  • Container and IaC: validates Compose generation, web/API/db separation, health checks, file-watch rebuilds, external URL logs, registration-first JSON auth, environment contracts, and Tailwind class-layer guidance.
  • Env and deploy: protects carbide health, carbide health env, carbide health runtime, checked-in deploy-script targets, guarded unknown targets, and the app-owned carbide deploy prod entrypoint.
  • Frontend: protects the Bun/React/Tailwind container, same-origin API proxy, lockfile install path, cache-busted browser assets, and dashboard app shell.
  • API: protects JSON auth responses, session cookies, /api/me, and Postgres-backed dashboard data.
  • CLI golden tests: protects scaffolding, deterministic generated files, and actionable command errors.
  • Security regression: protects cookies, CSRF, upload limits, SQL parameterization, and secret handling.
  • Documentation regression: keeps examples, extensionless links, redirects, and published docs aligned with framework behavior.
